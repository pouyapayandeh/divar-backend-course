# دیتابیس

یکی از نیازهایی که تقریبا همیشه توی پروژه‌ها باهاش مواجهیم ذخیره‌سازی اطلاعاته. برای رفع این نیاز از دیتابیس استفاده می‌کنیم. مفهوم دیتابیس یکی از مهم‌ترین چیزهاییه که توی برنامه‌نویسی تحت وب باهاش سر و کار داریم.

برای ذخیره‌ی اطلاعات توی دیتابیس مدل‌های مختلفی وجود داره اما پر استفاده ترین اون ها Relational DBئه. با دیدن این لینک می‌تونی با مفاهیم اصلی‌ای که سر و کار داریم آشنا بشی.

دیتابیس‌های مختلفی هستند که به زبان SQL کار می‌کنن. از بین این دیتابیس‌ها Postgresql رو بهتره که بیشتر باهاش آشنا بشیم چون توی بیشتر پروژه‌هامون داره استفاده میشه. برای همین بهتره که یه نگاهی به این لینک بندازی.

در کنار انواع مختلف دیتابیس‌های SQL دسته‌ی دیگه‌ای از دیتابیس‌ها هم هستن که اساسا برای برطرف کردن نیازهای دیگری به وجود اومدن به اسم NoSQL. برای اینکه فرق‌های بین این دو تا رو بهتر متوجه بشی این لینک‌ می‌تونه کمک بکنه.

همون‌طور که مشخصه خود دیتابیس‌های NoSql هم انواع مختلفی دارن که میشه توی چهار دسته‌ کلی‌تر قرارشون داد.

# اهداف

  • آشنایی با دیتابیس رابطه ای
  • یادگیری زبان SQL
  • وصل شدن به دیتابیس رابطه ای از طریق Python

دیتابیس رابطه ای (relational) یکی از پر استفاده ترین مدل دیتابیس هاست. برای کار با دیتابیس‌های رابطه‌ای از زبان SQL استفاده می‌شه.

# برای مطالعه

  • برای آشنایی با زبان SQL پیشنهاد می‌کنیم که این لینک رو به عنوان منبع بخونی
  • مجموعه‌ای از ویژگی‌ها هستن که با توجه به transaction‌های دیتابیس برای ما اهمیت ویژه‌ای دارن. این مجموعه ویژگی‌ها با کلمهٔ ACID نمایش داده میشن. پیشنهادمون اینه که با دیدن این لینک‌ها بیشتر با Atomicity, Consistency, Isolation, Durability آشنا بشی.

# تمرین

  1. به کمک psycopg2 یه جدول تولید کن که توی اون بتونیم مشخصات کاربران (نام، ایمیل، شماره تلفن و سن) رو ذخیره کنیم و ۱۰۰۰ تا کاربر به اون اضافه کن! برای این که کاربرامون یه خورده واقعی‌تر باشن می‌تونی از این ماژول استفاده کنی.
  2. یک جدول دیگه هم برای تسک‌ها بساز!‌ (عنوان، تخمین زمانی، ارزش، ددلاین و کاربر)
  3. یک کوئری بنویس که تسک‌ها رو به ترتیب ددلاین‌هاشون برگردونه.
  4. یک کوئری بنویس که جمع هزینه‌ی تسک‌هایی که برای هر کاربره رو حساب کنه.
  5. به جدولی که برای کاربرها ساختی یک فیلد is_active اضافه کن که مقدار پیش‌فرضش true باشه. (به همون جدول اضافه کن و دوباره از اول جدول جدید نساز)
  6. حالا یک کوئری بنویس که مقدار is_active همه‌ی کاربرانی که سنشون زیر ۱۸ ساله رو false کنه.
  7. دو کاربر می‌تونن با هم دوست باشن. یک جدول میانی درست کن که این روابط دوستی رو ذخیره کنه و تعدادی رابطه‌ی دوستی تست اضافه کن.
  8. یک کوئری بنویس که اسم دوستان کاربرانی که بالای ۱۸ سال دارن رو به صورت یکتا برگردونه.

# اهداف

  • آشنایی با Redis
  • آشنایی با چند ساختمان داده در ردیس
  • وصل شدن به ردیس از پایتون

ردیس یه جور دیتابیسه برای نگهداری از چند نوع ساختمان داده مختلف توی RAM. دلیل اینکه ما از ردیس استفاده می‌کنیم سرعت زیاد اونه چون اطلاعاتش رو روی RAM نگه می‌داره نه روی Disk.

# برای مطالعه

  • برای آشنا شدن با اینکه ردیس چه کارایی می‌تونه بکنه بد نیست یه نگاه خیلی سرسری به لیست دستورات ردیس داشته باشید که اینجا نوشته شدن.
  • پیشنهاد می‌کنیم برای کار با ردیس توی پایتون از این پکیج استفاده کنی.

# تمرین

  • فرض کنید یه بازی آنلاین داریم. می‌خوایم یه تابع بنویسیم که رکورد کاربرهای بازی رو ثبت کنیم این تابع یه str به عنوان نام کاربری و یه int به عنوان عدد رکورد از ورودی می‌گیره و روی ردیس ذخیره می‌کنه. یه تابع دیگه داریم که با گرفتن نام کاربری رکورد رو برمیگردونه.
  • با قابلیت‌های ردیس کاری کنید که هر رکورد فقط تا یک هفته معتبر باشه و بعد از یک هفته اتوماتیک حذف بشه
  • با استفاده از ساختمان داده درست توی ردیس تابعی بنویسید که بشه لیست ۱۰ رکورد برتر رو گرفت.
