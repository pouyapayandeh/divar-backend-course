# کوبرنتیز

ابتدا برای اینکه بدونی kubernetes چیه و چرا ازش استفاده می‌کنیم این لینک رو بخون و این ویدیو رو ببین. برای آشنا شدن با مفاهیم kubernetes می‌تونید از این لینک استفاده کنید.

برای بالا آوردن kubernetes به صورت لوکال از این لینک استفاده کنید. (در صورت نیاز از منتورت کمک بگیر)

در این قسمت می‌خوایم یک پروژه‌ی جنگو رو همراه تمام dependancyهاش روی kubernetes بیاریم بالا. برای شروع یک پروژه‌ی جنگو با REST-API بسازید که یک Endpoint با فعل GET داشته باشه که یک Hello World ساده برمی‌گردونه لازمه که توی این قسمت نکات زیر رو رعایت کنیم:

  • اول docker براش بسازیم
  • برای خود پروژه یه deployment با حداقل ۳ تا replica بیاریم بالا.
  • برای پروژه یه service روی port شماره ۸۰۰۰ بسازیم و بتونیم بهش دسترسی پیدا کنیم.

حالا فرض کنید redis و postgresql و nginxی برای پروژه می‌خوایم و می‌خوایم که از هر کدوم یه دونه روی kubernetes‌ بیاریم بالا و از طریق service به پروژه وصلشون کنیم.

  • برای پروژه‌مون readiness probe و liveness probe بذاریم یه جوری که منطقی باشه.
  • اگه دیتابیس داریم برای این که وقتی pod دیتابیسمون پاک می‌شه دیتای روش نپره یه فکری بکنیم (راهنمایی: local volume یا hostpath)
  • برای podهای پروژه و redis و postgresql استفاده از resourceهارو محدود کنیم. ینی کاری کنیم که نتونن از یه حدی بیشتر cpu و memory استفاده کنن.
  • فرض کنید پروژه‌ای داریم که دیتابیس داره و میخوایم فرایند بکاپ گیری رو ببریم رو کوبر. برای این کار یه job کوبر بنویس تا اسکریپت بکاپ گیری مون رو کوبر انجام بشه. ولی حواست به این موضوع باشه که فضای ذخیره سازی رو کوبر ممکن کم باشه و حجم دیتابیس ای که می خوایم بکاپ بگیریم زیاد باشه برای همین به این فکر کن که چطور این مشکل رو میشه حل کرد

برای انجام دادن کارای بالا خوبه که از خود سایت kubernetes استفاده کنید و سعی کنید بهترین راه حلی که پیدا می‌کنید رو انجام بدید.

deployment و service و … خودتونو به صورت فایل‌های yaml بسازید و داخل git پروژه قرار دهید طوری که یه نفر اگه یه kubernetes خالی رو تحویل بگیره بتونه با این فایل‌ها همه‌ی چیزایی که بالا گفتیم رو بیاره بالا.
