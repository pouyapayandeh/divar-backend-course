    # برای مطالعه

    • از اونجایی که برنامه‌مون قراره که تحت وب باشه و مرورگر و وب‌سایت و همه‌ی اینجور ابزارا به کمک شبکه و پروتکل http کار می‌کنن، خوبه که شهودی روشون داشته باشی. به خاطر همین از اینجا تا آخر بخش Header Fields رو بخون.
    • برای آشنا شدن بیشتر با مفاهیم DNS و لایه‌های پروتکل شبکه می‌تونی لینک‌های این و این و این رو ببینی.
    • خیلی وقتا برای کارهای مربوط به شبکه، نیاز میشه که سراغ ترمینال لینوکس بریم. از پیدا کردن IP خودمون یا گرفتن یک فایل از اینترنت تا تست کردن یه شبکه یا وصل شدن به یک سرور و حتی گوش کردن روی یک پورت. برای اینکه با این دستورها بیشتر آشنا بشی پیشنهاد می‌کنیم صفحه‌ی ۱۷۵ تا ۱۸۸ و ۱۹۶تا ۱۹۸ کتاب راهنمای لینوکس جیبی(linux pocket guide) رو بخونی.

    # تمرین

    1. آی‌پی سایت divar.ir رو پیدا کن.
    2. همه‌ی برنامه‌هایی که از پورت ۸۰ استفاده می‌کنن رو ببند.
    3. چک کن کدوم پورت‌های سیستمت بازه و برای چی؟
    4. با کمک ابزار Network مرورگرت، درخواست‌هایی که موقع رفتن به سایت divar.ir زده میشه رو نگاه کن و تمام اطلاعاتی که از سرور دیوار یا دستگاه خودت مشاهده می‌کنی رو بنویس. این کار رو در حالی که یک ویدیو از سایت آپارات رو نگاه می‌کنی هم انجام بده.
