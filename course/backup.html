    # بکاپ گیری

    یکی از کار های مهمی که باید انجام بدیم اینه که از دیتاهامون به صورت منظم بک‌آپ بگیریم و جایی نگه داری کنیم که اگر زمانی برای سرور و دیتاهای اصلی مشکلی پیش اومد یا از دسترس خارج شد بتونیم دیتاهامون رو برگردونیم.

    یکی از موردهای که باید بهش توجه کنیم اینه که خود بک‌آپ هامون رو کجا نگه داریم (قاعدتا کنار دیتابیس نگه نمی‌داریم) برای این‌که خودمون رو خیلی درگیر مشکلات ذخیره سازی و یا زیاد شدن دیتا های بک‌آپ‌مون نکنیم از فناوری‌های Object storage مثل (Amazon S3) استفاده می‌کنیم.

    # تمرین

    • یه minio-server بالا بیار.
    • تو این قسمت یه اسکریپت بنویس که بره از یک فایل توی یک آدرس مشخص هست بک‌آپ بگیره طوری که بک‌آپی رو که گرفته با minio-client رو minio-server بریزه.
    • یه سرویس با تایمر یا یه cronjob برای سیستمت بنویس که هر روز سر یه ساعت خاصی یا به صورت بازه‌های زمانی منظم اسکریپتی که نوشتی رو اجرا کنه.

    # برای مطالعه‌ی بیشتر

    اگر دیتا توی دیتابیس باشه اغلب لازمه که دیتاهای یک دیتابیس رو بک‌آپ بگیریم که می‌تونیم از دیتابیس دامپ بگیریم و فایل دامپ رو بک‌آپ بگیریم. برای نمونه توی postgres برای این کار از دستور pg_dump استفاده می‌کنیم.
