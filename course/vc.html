    # کنترل نسخه

    برای این‌که افراد یک تیم روی یک پروژه بتونن کار‌های مختلف رو جلو ببرن از ابزار‌های کنترل نسخه (version control) مثل گیت (Git) استفاده می‌شه. ویدیویی برای یاد‌گرفتن گیت آماده کردیم که توی این آدرس می‌تونی ببینیش.

    تو دیوار همهی پروژه های یک branch به اسم master دارن(خود git اولین branchی که می‌سازه به همین اسمه) و بعد برای هر feature که قراره یه پروژه اضافه بشه یه branch جدا درست می کنیم و بعد از آماده شدن اون یک merge request‌ برای اون branch می سازیم. برای این که بدونی این روند چجوری کار می‌کنه خوبه که Gitlab Flow رو بخونی.

    # تمرین

    1. دستورات زیر رو برو بخون و مسلط باش چون خیلی به کارت میان:

       Git [ stash | branch | checkout | rebase | commit | push | reset ]

    2. در مورد فایل gitignore مطالعه کن، با تنظیم کردن محتویات فایل gitignore فایلای نامربوط روی گیت فرستاده نمی‌شن.

    3. راجع به زبان markdown بخون چون فایل های readme رو تو پروژه های مختلف با این زبان نوشتیم.

    4. فرض کن وسط کاری روی برنچ A هستی و تغییرات اخیرت رو کامیت نکردی. کاری پیش میاد و مجبوری بری روی برنچ B کار کنی و کد‌هات روی اون برنچ رو پوش کنی و برگردی سر برنچ A. چه شکلی می‌تونی به کد‌هایی که زدی و کامیت نکردی دسترسی داشته باشی؟ (می‌تونی از دستور git stash استفاده کنی)

    5. پیدا کن که چه جوری می‌شه کد الان رو با کد مربوط به کامیت‌های قدیمی‌تر مقایسه کرد؟

    6. این مقاله یه سری اصول و استاندارد در مورد متن پیام کامیت‌ه که بهتره بخونی.
